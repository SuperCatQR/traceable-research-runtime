# 前端组件与主题色系统

- 状态：已完成 Demo 主题重构
- 范围：`web/src/demo.ts` 和 `web/src/demo.css` 的本地前端演示
- 目标：让浅色与深色主题使用相同的组件层级和交互含义，而不是为个别元素追加颜色覆盖规则

## 设计结论

研究工作区的视觉重点是阅读内容和可复核证据，不是导航或品牌色。因此蓝色只承担定位、链接、焦点和小型主操作；不作为长文本的背景。页面靠中性色表面层级区分区域。

主题保留两个：浅色 `mist` 与深色 `carbon`。不增加第三个主题，直到正式产品需要品牌或高对比变体。

## 组件分类

| 类别 | 组件 | 必需颜色角色 |
| --- | --- | --- |
| 应用壳层 | 页面画布、工作区、中央文档、侧栏、研究过程栏 | `canvas`、`shell`、`document`、`panel`、`border` |
| 导航与身份 | 品牌、研究列表、搜索框、账户入口、归档入口 | `nav-text`、`nav-muted`、`nav-hover`、`nav-selected`、`nav-marker` |
| 命令与输入 | 新研究、图标按钮、模型选择、编辑框、研究输入框、发送按钮 | `control`、`control-hover`、`field`、`focus`、`action`、`action-ink` |
| 阅读内容 | 问题、标题、正文、引文、比较块、建议列表 | `text-primary`、`text-secondary`、`text-muted`、`document`、`rule`、`citation` |
| 证据追溯 | 概览标签、覆盖统计、来源卡、审计记录、证据标记 | `trace-panel`、`trace-card`、`trace-rail`、`trace-selected`、`accent` |
| 反馈状态 | 已完成、研究中、错误、空状态、加载占位 | `success`、`info`、`danger`、`warning` 及各自的 `surface`、`border`、`ink` |
| 演示辅助 | 演示状态切换器、本地数据标记 | `utility-surface`、`utility-border`、`utility-text`、`utility-active` |
| 覆盖层 | 移动端抽屉遮罩 | `scrim` |

## 颜色使用规则

1. 长段正文永远使用 `document` 或 `panel` 表面，不能使用 `accent` 表面。
2. 蓝色只用于不超过单行高度的标记、图标、链接、焦点和主操作；主操作文字仅限短命令。
3. 选中导航项使用中性表面加蓝色左标，而不是蓝色底配反白文字。
4. 边界由 `border` 与相邻表面明度差共同建立；不依赖大面积纯白。
5. 每个反馈状态必须同时定义文字、边界和表面，禁止只换文字颜色。
6. 组件样式不直接写十六进制颜色，颜色只出现在两个主题 token 块中。

## 验收标准

- 浅色模式中，画布、导航、工作区、正文和研究过程栏可被肉眼区分，但正文不是刺眼的大面积纯白。
- 深色模式中，没有蓝底承载两行以上的反白正文。
- 同一类组件在两种主题下仍具有相同的优先级和状态含义。
- 切换主题时，输入框、来源卡、错误状态、加载状态、演示切换器与移动端抽屉均改变到对应主题。
- CSS 检查中，组件规则不保留主题相关的直接十六进制颜色。
