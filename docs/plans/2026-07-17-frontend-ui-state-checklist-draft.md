- 状态：草案，待审核

- 产品名称：Traceable Search
- 依据：[产品说明与页面导航](2026-07-17-frontend-product-and-navigation-draft.md)、
  [桌面与手机低保真线框图](2026-07-17-frontend-low-fidelity-wireframes-draft.md)、

- 范围：正式前端 MVP 的正常、加载、空、错误和恢复状态
- 用途：交互设计审核、组件状态设计、开发验收和端到端测试编号

## 目标

用户在任何时刻都应该知道三件事：

1. 当前看到的是有效内容、正在加载、没有内容，还是发生了错误。
2. 当前是否可以继续操作。
3. 如果不能继续，下一步应该做什么。

状态文案从用户视角描述结果，不暴露 HTTP、内部状态机、Research Brief、运行 ID 或模型原始错误。

## 四类状态定义

| 类别 | 定义 | 典型表现 | 不能混淆为 |
| --- | --- | --- | --- |
| 正常 `N` | 数据有效，用户可以阅读或执行当前允许的动作 | 页面内容、等待补充、研究运行中、已完成答案 | “什么都没发生”的默认占位 |
| 加载 `L` | 正在等待一次明确的读取或写入操作完成 | 骨架、按钮内加载、局部状态行 | 研究管线的虚构进度 |
| 空 `E` | 请求成功，但当前集合或内容确实为空 | 空状态文案 + 一个合适的下一步 | 加载失败或权限失败 |
| 错误 `X` | 请求、业务约束或数据契约失败 | 原位置错误 + 恢复动作 | 空列表、研究正常耗时 |

研究的 `clarifying`、`ready`、`running` 是正常业务状态。只有读取这些状态的 HTTP 请求本身未完成时，才属于加载状态。

## 状态呈现原则

### 状态就地处理

- 页面级问题在页面主体处理，例如启动失败、资源不存在。
- 表单问题贴近字段或表单动作处理，例如邮箱、API 地址、模型验证。
- 对话问题留在对应消息或轮次内处理，例如发送失败、研究失败。
- Trace 问题只出现在右侧栏，不覆盖或清空聊天正文。
- Toast 只用于不占据固定位置的短反馈，不能承载必须操作才能恢复的错误。

### 保留已有内容

- 首次读取使用 Loading；已有内容的刷新使用“旧内容 + 刷新状态”。
- 刷新失败保留旧内容并标记“可能不是最新”，不退回空状态。
- 消息发送失败保留原文，模型配置保存失败保留所有非敏感字段。
- 切换对话、关闭侧栏或请求失败时保留合理的滚动位置。

### 加载反馈

- 少于约 `200ms` 的普通读取不强制闪现骨架，超过后再显示，避免闪烁。
- 用户主动提交后立即反馈，不等待 `200ms`：按钮进入加载状态，重复提交被阻止。
- 骨架只模拟稳定布局，不模拟具体文字，不使用跳动宽度。
- 不展示没有服务端依据的百分比、搜索词、来源数量或阶段轮播。
- 同一区域同时最多有一个主要加载信号。

### 空状态

- 空状态是成功结果，语气中性，不使用“出错了”“暂无数据”等含糊表达。
- 每个空状态最多提供一个主要动作；如果没有合理动作，只说明现状。
- 搜索无结果必须保留搜索框，并提供清除搜索条件的动作。

### 错误文案

错误由三部分构成：

```text
发生了什么：模型连接验证失败
用户可理解的原因：请检查 API 地址、密钥和模型 ID
恢复动作：[再次验证]
```

- 不使用“未知错误”“Something went wrong”“请稍后再试”作为唯一信息。
- 不向用户展示堆栈、HTTP 状态码、内部错误枚举或上游响应正文。
- 同一个动作保持同名，例如按钮“再次验证”，成功反馈“连接已验证”。

## 状态优先级

同时满足多个状态时按以下顺序决定主界面：

1. 登录失效或账户切换：先隐藏账户数据并进入重新登录。
2. 当前路由资源不存在或不可访问：显示统一的未找到页面。
3. 首次模型配置门槛：没有可用模型时进入模型配置。
4. 页面首次加载或页面级错误。
5. 页面正常/空状态。
6. 局部加载、局部错误和后台刷新提示。

局部状态不能覆盖更高优先级状态。例如登录失效后，不继续显示“正在加载研究概览”。

## 正常状态清单

| ID | 表面 | 触发条件 | 界面表现 | 允许动作 |
| --- | --- | --- | --- | --- |
| `N-AUTH-01` | 登录 | 未登录且表单可用 | 登录标签选中，字段为空或保留当前输入 | 登录、切换注册 |
| `N-AUTH-02` | 注册 | 注册标签选中 | 显示名称、邮箱、密码字段 | 创建账户、切换登录 |
| `N-WORK-01` | 研究工作区 | 已登录、有模型、有研究列表 | 左侧最近研究 + 恢复的当前对话 | 新研究、切换对话、打开归档/设置 |
| `N-CONV-01` | 空研究对话 | 对话创建成功但尚无轮次 | 标题、可用输入区、“写下需要查证的问题” | 发送第一条研究问题 |
| `N-TURN-01` | 等待补充 | `clarifying + awaiting_message` | 显示模型的自然追问或理解 | 补充、纠正当前问题 |
| `N-TURN-02` | 正在理解 | 当前提交未返回，或加载到 `clarifying + thinking` | 待发送消息/已有对话 + 固定“正在理解”状态 | 当前轮不可再次提交 |
| `N-TURN-03` | 研究已安排 | `ready` | “即将开始研究”或等价紧凑状态 | 查看历史内容；输入禁用 |
| `N-TURN-04` | 研究运行中 | `running` | “正在检索、锁定快照并核验来源” | 查看历史内容；输入禁用 |
| `N-TURN-05` | 研究完成 | `completed + answer` | Markdown 答案、证据轨道、来源入口 | 打开来源/概览、创建下一轮 |
| `N-TURN-06` | 研究已取消 | `cancelled` | `本轮研究已取消` | 创建下一轮 |
| `N-CONV-02` | 多轮对话 | 已有多个终结轮次 | 按时间顺序展示完整对话，输入区针对下一轮 | 阅读、追问、切换轮次概览 |
| `N-TRACE-01` | 研究概览 | Summary 成功且至少有一个区段 | 问题理解、检索覆盖、来源、综合/失败摘要 | 切换轮次、打开审计 |
| `N-TRACE-02` | 审计详情 | Audit 首屏有事件 | 阶段筛选、事件列表、分页入口 | 筛选、加载更多、回到概览 |
| `N-MODEL-01` | 可用模型列表 | 至少一个活跃配置 | 默认、验证状态、名称和模型 ID | 添加、编辑、验证、设默认、归档 |
| `N-MODEL-02` | 模型编辑 | 打开一个活跃配置 | 当前非密钥字段；Key 留空表示保留 | 保存、验证、设默认、归档 |
| `N-ARCH-01` | 已归档研究 | 归档列表非空 | 按归档时间排列，显示标题、轮次、原模型 | 搜索、恢复 |
| `N-ARCH-02` | 已归档模型 | 归档模型列表非空 | 只读摘要和恢复动作 | 恢复 |
| `N-MOBILE-01` | 手机研究列表 | 用户打开左侧抽屉 | 当前研究高亮，正文在遮罩后保留 | 选对话、新研究、打开归档 |
| `N-MOBILE-02` | 手机研究过程 | 用户打开概览/审计 | 全屏研究过程，顶部关闭按钮 | 切换标签/轮次、关闭 |

### 正常状态不应出现的内容

- 主聊天不展示 Brief、查询列表、隐藏推理、模型 API 地址或运行 ID。
- `ready/running` 不展示“第 2/5 步”等假阶段。
- 已完成答案不默认展开整个审计记录。
- 手机端不同时打开研究列表和研究过程。

## 加载状态清单

| ID | 表面 | 触发条件 | 界面表现 | 完成后 | 禁止行为 |
| --- | --- | --- | --- | --- | --- |
| `L-BOOT-01` | 应用启动 | `GET /auth/me` 未完成 | 全屏品牌标记 + 单一加载指示 | 登录页、首次模型配置或工作区 | 闪现上次账户内容 |
| `L-AUTH-01` | 登录/注册 | 提交认证请求 | 主按钮原位加载，当前表单禁用 | 进入下一状态或显示表单错误 | 切换标签、重复提交 |
| `L-WORK-01` | 模型与对话列表 | 登录后并行首次读取 | 保留工作区框架；列表使用稳定骨架 | 正常/空/错误 | 自动创建对话 |
| `L-MODEL-00` | 模型设置首次读取 | 进入设置且没有缓存 | 保留设置页标题；列表和编辑区使用稳定骨架 | 正常/空/错误 | 显示空白表单供提交 |
| `L-ARCH-00` | 已归档研究首次读取 | 进入归档页面且没有缓存 | 保留页面标题和搜索位置；列表使用稳定骨架 | 正常/空/错误 | 暂时显示“没有已归档研究” |
| `L-CONV-01` | 创建新研究 | 创建对话请求 | “新研究”按钮加载；当前对话仍可阅读 | 打开新空对话 | 清空当前正文后等待 |
| `L-CONV-02` | 对话详情 | 选择未缓存对话 | 中间正文骨架，左侧选择保持 | 正常/空/错误 | 让旧对话标题配新正文 |
| `L-CONV-03` | 对话后台刷新 | 轮询或重新聚焦 | 保留正文；仅显示低强调刷新状态 | 静默更新或 stale 错误 | 全屏骨架、跳到顶部 |
| `L-MSG-01` | 新研究问题 | `POST /turns` 未完成 | 立即显示用户消息“发送中” + 正在理解 | 用服务端 Turn 替换 | 丢失消息、自动生成第二次请求 |
| `L-MSG-02` | 补充消息 | `POST /messages` 未完成 | 补充消息“发送中” + 输入禁用 | 替换当前 Turn | 继续编辑 revision 已锁定的消息 |
| `L-TRACE-01` | 首次概览 | 打开右栏并请求 Summary | 右栏内部骨架，聊天不变 | 概览/空/错误 | 阻塞输入区 |
| `L-TRACE-02` | 首次审计 | 第一次切到审计 | 筛选框位置固定，事件区骨架 | 列表/空/错误 | 请求前加载 L3 原始数据 |
| `L-TRACE-03` | 加载更多审计 | 使用 `next_cursor` | 保留已有事件，底部按钮原位加载 | 追加事件或分页错误 | 清空已加载事件 |
| `L-MODEL-01` | 保存模型 | 创建或更新请求 | 保存按钮加载，表单暂时只读 | 已保存待验证/成功/错误 | 同时发起第二次保存 |
| `L-MODEL-02` | 验证模型 | Verify 请求 | “验证连接”原位加载 | 已验证或验证失败 | 把它显示成全局页面加载 |
| `L-MODEL-03` | 设置默认 | Default 请求 | 目标行局部加载 | 默认标记转移或错误 | 禁用整个设置页 |
| `L-ARCH-01` | 归档资源 | 用户确认后请求 | 触发行局部加载，资源仍在原位置 | 移到归档列表或错误 | 提前从页面移除资源 |
| `L-RESTORE-01` | 恢复资源 | Restore 请求 | “恢复”按钮加载 | 移到活跃列表或错误 | 同一项重复恢复 |

### 业务运行状态与网络加载的区别

| 场景 | 正确分类 | 原因 |
| --- | --- | --- |
| 等待 `POST /turns` 返回 | `L-MSG-01` | 浏览器正在等待一次请求 |
| 服务端已返回 `ready` | `N-TURN-03` | 已知研究已安排，是有效业务状态 |
| 服务端返回 `running` | `N-TURN-04` | 已知研究正在运行，不是页面 Loading |
| 轮询 `GET conversation` | `L-CONV-03` + 保留 `N-TURN-04` | 网络刷新不能替换已知业务状态 |

## 错误状态清单

### 应用、认证与路由

| ID | 触发条件 | 界面位置 | 建议文案 | 恢复动作 |
| --- | --- | --- | --- | --- |
| `X-BOOT-01` | 启动请求无响应/500 | 全屏启动错误 | `无法打开研究工作区` | `重试` |
| `X-AUTH-01` | `invalid_credentials` | 登录表单 | `邮箱或密码错误` | 修正后 `登录` |
| `X-AUTH-02` | 注册字段错误 | 对应字段 | 使用服务端公开字段说明 | 修正字段 |
| `X-AUTH-03` | `email_already_registered` | 邮箱字段 | `该邮箱已注册` | `前往登录` |
| `X-AUTH-04` | 认证请求网络/500 | 表单底部 | `暂时无法连接服务` | `重试` |
| `X-AUTH-05` | `authentication_required` | 全局状态切换 | `登录已失效，请重新登录` | `登录`；同账户恢复内存草稿 |
| `X-ROUTE-01` | `not_found` 或无效路径 | 页面主体 | `未找到请求的内容` | `返回研究工作区` |
| `X-OFFLINE-01` | 浏览器离线或连续网络失败 | 顶部持久状态条 | `网络连接已断开` | 自动监听恢复 + `重试` |

| `X-CONV-06` | 归档请求失败 | 归档确认区 | `研究对话未归档` | `重试` |
| `X-CONV-07` | 归档列表首次加载失败 | 归档页面主体 | `无法读取已归档研究` | `重试` |
| `X-CONV-08` | 恢复请求失败 | 归档目标行 | `研究对话未恢复` | `重试` |
| `X-CONV-09` | 原模型已归档 | 恢复对话流程 | `原模型配置已归档` | `恢复模型` 或 `选择其他模型` |

### 消息、轮次与研究结果

| ID | 触发条件 | 界面位置 | 建议文案 | 恢复动作 |
| --- | --- | --- | --- | --- |
| `X-MSG-01` | 问题/消息字段无效 | 输入区上方 | 服务端公开字段说明 | 保留原文并修正 |
| `X-MSG-02` | 消息网络失败且结果未知 | 待发送消息下方 | `消息状态尚未确认` | 先刷新；再用原 Key 重试 |
| `X-MSG-03` | `idempotency_request_in_progress` | 待发送消息下方 | `消息仍在处理中` | 自动使用同一 Key 稍后重试 |
| `X-MSG-04` | `dialogue_revision_conflict` | 待发送补充消息下方 | `对话已更新，请查看最新消息后重新发送` | 刷新，保留文本供确认 |
| `X-MSG-05` | `turn_not_accepting_messages` | 当前轮输入区 | `这轮研究已进入下一状态` | 刷新对话 |
| `X-MSG-06` | 模型配置在提交期间变化 | 待发送消息下方 | `研究使用的模型已变更` | 刷新后手动重发 |
| `X-TURN-01` | Dialogue `failed` 但可补充 | 当前轮系统消息 | 使用服务端 `failure` 或 `模型暂时无法继续理解该问题` | 补充或重新表达 |
| `X-TURN-02` | Turn `failed` | 当前轮结尾 | `本轮研究未完成` | `研究概览`、创建下一轮 |
| `X-TURN-03` | 轮询连续失败 | 当前运行状态旁 | `暂时无法刷新研究状态` | 手动刷新；保留 ready/running |
| `X-SOURCE-01` | 来源 URL 不是 HTTP(S) | 来源项 | 来源标题显示但不可点击 | 无；记录客户端诊断 |
| `X-SOURCE-02` | 外部来源打开后不可访问 | 浏览器外部页面 | 本应用不猜测外站结果 | 用户返回后可看研究概览 |

研究失败是本轮的终结业务结果，不显示全局 Toast，不清空此前成功轮次，也不自动再次研究。

### 数据契约错误

| ID | 触发条件 | 用户表现 | 客户端处理 |
| --- | --- | --- | --- |
| `X-CONTRACT-01` | 未知 Turn/Dialogue status | `应用版本与服务不兼容，请刷新` | 禁止提交，记录原始枚举名 |
| `X-CONTRACT-02` | `completed + answer=null` | `答案暂时无法读取` | 提供刷新；不显示空答案 |
| `X-CONTRACT-03` | `clarifying + dialogue=null` | `对话状态暂时无法读取` | 禁用输入并刷新详情 |
| `X-CONTRACT-04` | 成功响应缺少必需字段 | 当前表面显示无法读取 | 记录接口、字段和 request context |

契约错误不使用服务端 `message` 掩盖，也不能自动转换成空状态。

## 后台刷新与旧数据状态

| ID | 场景 | 界面规则 |
| --- | --- | --- |
| `S-STALE-01` | 列表刷新中 | 旧列表可操作，顶部显示低强调刷新状态 |
| `S-STALE-02` | 对话轮询中 | 旧正文可阅读，输入权限仍由最后已知状态决定 |
| `S-STALE-03` | 刷新失败但有旧数据 | 保留旧数据并标记“可能不是最新”，提供刷新 |
| `S-STALE-04` | 重新联网 | 自动刷新当前页面及活动轮次，成功后移除离线条 |

如果最后已知状态是 `ready/running`，刷新失败时输入区继续禁用，防止基于未知新状态创建冲突轮次。

## 成功反馈

成功状态通常直接体现在页面中，避免为每个动作弹 Toast：

| 动作 | 成功反馈 |
| --- | --- |
| 登录/注册 | 进入下一页面，不显示 Toast |
| 创建研究 | 打开新对话 |
| 重命名 | 标题编辑器关闭，显示新标题 |
| 保存模型 | 表单显示服务端返回内容；首次流程继续验证 |
| 验证连接 | 状态改为“已验证” |
| 设为默认 | 默认标记移动到目标配置 |
| 归档 | 资源离开活跃列表，短暂显示“已归档” |
| 恢复 | 资源离开归档列表；研究对话恢复后打开 |

只有用户可能错过结果且页面没有明显变化时，才使用短暂成功 Toast。

## 手机端状态规则

- 页面级状态占满当前页面；局部错误保留在当前抽屉或全屏移动层内。
- Toast 位于底部输入区和安全区域上方，不遮挡发送按钮。
- 键盘打开时，表单错误必须能滚动到对应字段；焦点移到第一个错误字段。
- 研究列表加载失败不关闭抽屉；概览加载失败不跳回聊天。
- 离线条、登录失效和页面级错误不能依赖鼠标悬停获取说明。
- 打开新的全屏状态层时，屏幕阅读器焦点进入该层；关闭后返回原触发按钮。

## 可访问性与播报

| 状态 | 建议语义 |
| --- | --- |
| 页面首次加载 | 页面保持可识别标题；骨架 `aria-hidden=true` |
| 写操作进行中 | 按钮 `disabled` 或 `aria-disabled`，按钮名称保持可理解 |
| 研究运行中 | 非打断式 `role=status`，状态变化时播报一次 |
| 表单字段错误 | 字段 `aria-invalid=true`，通过 `aria-describedby` 关联错误 |
| 重要错误 | `role=alert`，避免同一错误在多个区域重复播报 |
| 后台刷新失败 | 非打断式状态提示，不抢走焦点 |
| 空状态 | 普通内容，不使用 alert |

加载动画必须遵守 `prefers-reduced-motion`，停止动画后仍保留文字状态。

## 验收检查表

- [ ] 每个页面都有 Normal、首次 Loading、Empty 和首次 Error。
- [ ] 每个带写操作的控件都有 submitting、成功和失败状态。
- [ ] 已有数据刷新时不会退回首次骨架或空状态。
- [ ] 所有错误都有明确位置和至少一个合理恢复路径，或明确说明不可恢复。
- [ ] 所有用户输入在可恢复错误后仍然存在。
- [ ] 研究 `ready/running` 不因轮询失败被前端改写为失败。
- [ ] Empty 只由成功响应产生，`404/500/network` 不进入 Empty。
- [ ] Trace 错误不影响聊天正文和输入区。
- [ ] 登录失效不会暴露上一账户数据，草稿只对同一账户在内存中恢复。
- [ ] 桌面和手机状态具有相同语义，只有布局不同。
- [ ] 错误处理使用稳定 code，不解析中文 message 做逻辑分支。
- [ ] 状态文案不包含内部术语、HTTP 状态码或虚构进度。

## 待审核决策

1. **首次读取 Loading 延迟**
   - 建议：约 `200ms` 后再显示骨架，用户提交立即反馈。
2. **刷新失败时保留旧数据**
   - 建议：保留并标记“可能不是最新”，只有首次加载失败才替换整个内容区。
3. **无来源答案的级别**
   - 建议：作为明显但非阻断的警告，仍允许阅读答案和查看研究概览。
4. **研究失败的恢复方式**
   - 建议：不提供“重跑”按钮；允许查看概览并创建下一轮，避免无幂等语义的自动重复研究。
5. **离线表现**
   - 建议：使用持续顶部状态条；恢复网络后自动刷新当前页面。
6. **成功 Toast**
   - 建议：仅用于归档等页面变化可能不够明确的动作，其他成功通过界面本身表达。
7. **发送失败的消息保留方式**
   - 建议：消息留在原位置并标记失败，重试复用同一个 `Idempotency-Key`。
8. **契约错误处理**
   - 建议：禁止相关写操作、提供刷新，并记录诊断；不能降级成空状态继续操作。
