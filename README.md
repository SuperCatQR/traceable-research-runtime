# Traceable Research

> 可溯源、可审计的 LLM 问答系统——让答案的来源可控，让可信度能被第三方机构评估。

**当前进展**：web-search 完成度 60%。

## 核心理念

大模型的回答常是黑箱：所依据的材料不明、推理过程不可查、可信度无从评估。本项目把这三件事变成可控、可查、可评估的工程问题。

- **可溯源**：每一条回答都能追溯到它所依据的材料。
- **可审计**：回答所用的知识边界由人工定义，第三方机构可据此评估其可信度。
- **来源可控**：模型思考时面对的场域、本体、认识与目的，均可人为设计。

## 两条主线

### database-search（数据库检索）

从设计上保证回答建立在**人工编辑的事实**之上——材料经过筛选，带有明确的编辑立场。它把模型思考时的四个维度交给人来定义：

- **场域背景**：回答所处的背景与语境
- **本体**：这个场域里"存在什么"
- **认识**：模型可调用的材料边界
- **目的**：回答要达成的目标

可以在数据库中假设一整套"某领域现状"，人为设定 AI 思考时所处的完整认知环境。因为知识边界完全由人工构建且可追溯，整个系统天然可审计。面对监管压力时，最直接的应对就是把数据库完全公开。

一切效果取决于**数据库质量**，而数据库本身用自然语言文档撰写即可，门槛不高。预计每个领域的数据库设计可支撑 1–2 个岗位。

### web-search（网络检索）

搜索结果可溯源，每条结论都能回溯到具体网页来源，供用户核验。它有两种用法：

- **辅助数据库设计**：为 database-search 的知识构建提供有据可查的原始素材。
- **独立工具**：不依赖预置数据库，实时检索并给出带出处的回答。单作为一个可溯源的 web-search 工具使用，本身就成立，也很不错。

## 数据库校准

数据库质量决定一切，因此需要一套校准机制。暂时设想的方式：用一个旗舰模型（如 fable5）向系统发问、挑刺，系统性地审查数据库的准确性与漏洞。校准不能全交给模型，关键判断仍需人工参与。

## 使用场景

场景上限取决于数据库质量，可从细分领域起步：

- **低门槛问答**：下载客户端或网页注册即可对话，开箱即用，省去大量前置工作（可作为付费入口）。
- **领域培训**：定向灌输特定内容，培养用户采用。
- **法律咨询**：从劳动法、民法等细分领域切入，逐步扩展（要求数据库设计得足够扎实）。
- **教育培训**：天然适配各类教学与培训场景。

## 目的

**知识共产**。项目一旦部署完成，能方便很多人。

## 商业模式

要落地就得考虑可持续的商业模式，才能健康推进。在没有监管压力时，可以不向用户告知完整的推理链条，收信息差的钱；项目完全可以作为一个或多个细分领域的工具。

## 技术栈

DeepSeek v4-pro + v4-flash 即可满足全部设计需求。

## 发展倾向

未来若构建领域数据库，倾向于将**基本生活领域**的部分开源。
