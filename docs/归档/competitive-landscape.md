# 竞品格局：可溯源对话检索

> 目的：厘清本项目在"可溯源对话式搜索"赛道中的位置，避免误判"无人做过"。
> 结论先行：**"对话搜索 + 给出处"不新，甚至拥挤；本项目的独特点在"可审计证据链 + 受控检索流 + 极低成本"三者组合。**

## 二、本项目的差异化（勿在迭代中丢失）

1. **逐字快照 + content_hash 存档**
   Perplexity 给活链接，明天可能改/删，引的"原文"不可复现；Anthropic 引文只活在单次请求上下文。
   本项目 `source_ref` 挂在存档快照 + 哈希上，是**可复现、可审计的证据链**。
   区别：可溯源=给指路；可审计=给证物。

2. **受控检索流，非模型自治**
   实测 v4pro 会直接凭记忆答、不调工具；Perplexity 亦由模型决定搜不搜。
   本项目架构层强制 `搜 → 选 → 读`，模型无"偷懒凭记忆"的选项。

3. **极低成本定位**
   Perplexity Pro $20/月、Max $200/月，底层烧 frontier 模型。
   本项目命题：**廉价模型 + 结构化检索追平贵模型**。这是它们的反面（它们卖"贵模型 + 便捷"）。

## 三、测试实证支撑（截至本文档）

- flash（弱模型）：带 MCP 碾压不带 → 检索对弱模型增益巨大。
- v4pro（强模型）：带不带 MCP 答案几乎一致且都对 → 对强模型，"可溯源"卖点弱（它自己就对）。
- 推论：**MCP 事实增益与模型强度成反比**。故本项目杀手锏必须落在差异点 1（证物链）与 3（廉价追平），而非单纯"更准"。
- 正确对比协议应为 **flash+MCP vs v4pro 纯记忆**（验便宜方案能否追平贵方案），而非 v4pro 自比。
